PRÁCTICA 4: DESENCRIPTACIÓN DE CONTRASEÑAS (FUERZA BRUTA DIRIGIDA)

El Reto: Romper un Hash SHA-256

En esta práctica hemos desarrollado un script en Python para descifrar una contraseña a partir de su huella digital criptográfica (hash). El objetivo era recuperar el texto plano del siguiente hash:

4a00382efb5acb17c52a699364e65fc0189a4235e4fb467eb085cfa565cfeaf4

Inteligencia previa: Para no hacer un ataque de fuerza bruta a ciegas (que tardaría años), partimos de tres pistas clave sobre la contraseña original:

  • Tiene exactamente 7 caracteres.
  • Solo utiliza letras minúsculas.
  • Contiene el patrón exacto cr7 en alguna posición.

Código Desarrollado (Python)

Este es el código conciso y optimizado que creamos en Visual Studio para realizar el ataque. Destaca la inclusión de la letra "ñ" para abarcar contraseñas en español:

import hashlib, itertools

target = "4a00382efb5acb17c52a699364e65fc0189a4235e4fb467eb085cfa565cfeaf4"
letras = "abcdefghijklmnopqrstuvwxyzñ" # Incluimos la ñ por si acaso

for c in itertools.product(letras, repeat=4):
    base = "".join(c)
    for i in range(5):
        intento = base[:i] + "cr7" + base[i:]
        if hashlib.sha256(intento.encode()).hexdigest() == target:
            print(f"¡Encontrado!: {intento}")
            exit()

¿Cómo funciona nuestro ataque?

  • Librerías clave: Importamos hashlib (para generar los hashes SHA-256) e itertools (para la combinatoria).
  • Variables iniciales: Definimos el hash que queremos romper en target y nuestro alfabeto en letras, asegurándonos de incluir la "ñ" para cubrir caracteres del español.
  • Reducción de combinaciones: Como sabemos que la clave tiene 7 letras e incluye "cr7", solo nos faltan por adivinar 4 letras. itertools.product(letras, repeat=4) genera todas las combinaciones de esas 4 letras. Almacenamos cada combinación en la variable base.
  • Deslizamiento del Patrón: Mediante el bucle for i in range(5), tomamos cada combinación generada en base y vamos insertando la cadena "cr7" en todas las posiciones posibles para formar nuestro intento.
  • Comprobación y Éxito: Cada intento se convierte a bytes (.encode()), se hashea a SHA-256 y se pasa a formato hexadecimal (.hexdigest()). Si es exactamente igual a nuestro target, el programa imprime el descubrimiento y usa exit() para detenerse instantáneamente.

Conclusión de la práctica

  • Esta práctica demuestra que los algoritmos de encriptación pueden ser robustos matemáticamente, pero la seguridad real depende de la complejidad de la contraseña.
  • Si un atacante tiene información de contexto sobre cómo creamos nuestras contraseñas (por ejemplo, saber que usamos aficiones como "cr7"), el tiempo de computación para vulnerar el sistema pasa de miles de años a apenas unos segundos.
